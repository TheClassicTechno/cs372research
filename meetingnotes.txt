1) Big question: agent + evaluation order vs concurrency
whether the agent work and evaluation should happen in order (sequential) or concurrently.

the overall system should be concurrent so teams don’t block each other;  created early PRs to reduce “waiting on dependencies.”


2) Proposed mental model: simulation loop (Theodore)
• Simulation environment runs through a predetermined sequence of decision points (“cases”).
• A case contains:
qualitative inputs (e.g., financial reports, news articles, other text signals)
price/market movement data (structured numeric signals)
• At each decision point:
simulation invokes the agent system
agent returns a trading action: buy / sell / hold, plus amount/size
simulation applies that decision to the portfolio
simulation then advances time until the next decision point
• At the end:
produce outputs like portfolio performance metrics
capture reasoning traces / transcript data to feed into evaluation/scoring

3) Single-stock vs multi-stock scope
Initial framing was one stock ticker with sequential cases.
 noted it can be multi-stock and “easily extensible.”

Theodore agreed multi-stock is possible later, but flagged that stock selection itself becomes a separate experimental design challenge (data volume + selection criteria).


4) Early dataset / stock choices
• Discussion leaned toward starting small:
one or two stocks (fast to get a full loop working)
then scale up after debugging and interface stabilization
• Suggested direction:
start with tech stocks (more abundant news coverage)
possibly mix stable (e.g., juli mentioned Microsoft  mentioned as “more stable”) + less stable stock for contrast

reduce ambiguity around how components interact

6) What exactly is “passed around”?

what is the object exchanged between simulatio agent

7) Debate / multi-agent behavior: inside agent or separate?
• Veljko asked:
is the debate (natural language back-and-forth) built into the platform/data model, or separate?
• Theodore’s answer:
debate is internal to the agent system

early work is mostly handling a text corpus (debate + reasoning chain) before sophisticated financial scoring
