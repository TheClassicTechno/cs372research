TASK 3 (HIGH PRIORITY): Multi-Ticker Portfolio Allocation
------------------------------------------------------------------------
STATUS: Not started. CRITICAL gap identified by Veljko (Slack 11:53 PM).

WHY: Currently the simulation presents ONE ticker per case to the agent.
     Real portfolio allocation requires the agent to see ALL tickers
     simultaneously and decide how to allocate across the universe.

PROBLEM:
  - case_loader loads cases per-ticker (NVDA/2025_Q1.json, KO/2025_Q1.json, etc.)
  - Runner iterates cases one at a time (simulation/runner.py:114)
  - Agent makes atomic buy/hold/sell per ticker, not portfolio-level allocation
  - This means no real portfolio optimization happens

STEPS:
  1. Read how Observation.universe is used in multi_agent/graph.py:
     - The graph already supports multi-ticker universe in the observation
     - Market prices dict can hold multiple tickers

  2. Option A: Merge same-quarter cases into one multi-ticker observation
     - Modify build_observation() in simulation/feature_engineering.py
     - Combine StockData from multiple Case objects into one Observation
     - Merge case_data items from all tickers for that quarter
     - This is the cleanest approach for portfolio-level debate

  3. Option B: Modify runner to batch cases by quarter
     - Group templates by quarter in _run_episode()
     - Build one combined Case per quarter with all tickers
     - Agent debates once per quarter, decides allocation across all tickers

  4. DISCUSS with team first — this is an architecture decision
